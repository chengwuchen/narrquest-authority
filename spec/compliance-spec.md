## 1. Scope
This specification defines compliance obligations derived from Failure Specification.

## 2. Upstream Authority
This document is subordinate to `failure-spec.md`.
In case of conflict, Failure Specification SHALL prevail.

## 3. Compliance Requirements
- CR-1: Any Run MUST record failure observations according to Failure Class taxonomy.
- CR-2: No downstream document may redefine failure semantics.
- CR-3: Editorial content MUST defer to normative specs.

## 4. Non-Compliance Handling
Non-compliance does not invalidate the system, but MUST be recorded.
